### 模块1：首页总览大屏

#### 核心定位

平台核心入口，轻量化展示全量数据的核心洞察，直观体现可视化技术的叙事能力，引导用户进入细分模块深度探索。

#### 技术落地与设计要点

1. 核心指标卡片化展示，用极简的可视化方式呈现数据核心统计特征，符合「少空间、少图形、短时间、多信息」的设计原则；
2. 长三角市域矢量地图为核心视觉主体，叠加夜间灯光栅格数据做底层渲染，实现科学可视化与信息可视化的融合；
3. 2011-2022年时序变迁动画，用拖尾轨迹展示数据的时空演化特征，实现「时间换取空间」的可视化叙事；
4. 核心模块快速入口，支持用户一键跳转至对应技术模块，降低探索门槛。

#### 核心可视化表达

核心指标卡片、时空分布动态地图、数据洞察摘要、模块导航入口。

### 模块2：数据预处理与质量评估模块

#### 核心定位

#### 技术落地与设计要点

1. 多模态数据目录树可视化，展示原始数据的结构、字段、来源、时间范围、空间覆盖，支持用户查看原始数据详情；
2. 预处理全流程可视化，分步骤展示空间匹配、缺失值处理、异常值处理、标准化变换、变量选择的过程与效果，每个步骤均提供处理前后的数据分布对比；
3. 数据质量评估体系，从完整度、准确性、一致性、时效性四个维度量化数据质量，可视化展示评估结果；
4. 交互式预处理工具，支持用户自定义调整预处理参数，实时查看处理效果，理解不同预处理方法对数据的影响。

#### 核心技术落地

整齐数据规范、缺失值/异常值处理、数据变换、变量选择、数据质量评估。

### 模块3：EDA全流程探索模块（平台核心）

#### 核心定位

平台技术核心模块，完整落地探索性数据分析的全流程方法论，实现无先验假设下的交互式数据探索，覆盖课程中所有EDA核心技术。

#### 技术落地与设计要点

1. **描述性统计子模块**：自动化计算全指标的基础统计量，用箱线图、小提琴图、直方图展示数据分布特征，支持单指标/多指标、单城市/多城市、单年度/多年度的对比分析；
2. **时序特征探索子模块**：用时序折线图、面积图、增长率柱状图展示指标的时序变化趋势，自动识别拐点、突变点、周期特征，支持多指标同比对比；
3. **空间特征探索子模块**：用分级设色地图、热力图展示指标的空间分布特征，通过莫兰指数、LISA集聚图分析空间相关性，可视化呈现高-高集聚、低-低集聚区域；
4. **变量关联分析子模块**：用散点图矩阵、相关性热力图展示变量间的关联强度，支持用户自定义选择变量，生成拟合线、置信区间，识别线性/非线性关系、同方差/异方差特征；
5. **因子重要性分析子模块**：用熵权法、DOE均值图量化指标贡献度，生成因子重要性排序，用水平柱状图、雷达图可视化展示，支持用户自定义赋权方法；
6. **数据降维探索子模块**：实现PCA主成分分析，可视化展示陡坡图、累计方差贡献率，支持用户自定义主成分数量，生成2D/3D降维散点图，实现高维数据的低维可视化探索；
7. **抽样分析子模块**：实现简单随机抽样、分层抽样、系统抽样的交互式操作，支持用户自定义抽样方法、样本容量，实时计算样本质量（KL信息量），对比样本与总体的分布差异，验证抽样代表性。

#### 核心技术落地

EDA全流程方法论、描述性统计、分布探索、变量关系分析、因子重要性判断、PCA降维、抽样分析、异常值检测。

### 模块4：多模态数据融合可视分析模块

#### 核心定位

平台特色技术模块，落地课程中科学可视化、信息可视化、可视分析三大分支的融合应用，实现多模态数据的跨维度关联与可视表达。

#### 技术落地与设计要点

1. **四大模态数据分栏可视化**：
   - 结构化统计数据：用统计图表实现信息可视化表达；
   - 遥感生态数据/夜间灯光数据：用栅格渲染实现科学可视化表达；
   - 专利文本数据：用词云、网络图实现非结构化数据的可视化表达；
2. **跨模态特征关联可视化**：用联动视图、桑基图、相关性热力图，可视化展示不同模态特征间的关联关系，实现「单模态特征变化→多模态关联波动」的联动探索；
3. **多模态融合指数可视化**：完整展示多模态特征融合的全流程，从特征提取→权重计算→融合指数生成，全步骤可追溯、可调整，用地图、时序图展示融合指数的时空分布特征；
4. **多视图协调关联设计**：所有模态的视图实现联动筛选、联动高亮、联动下钻，用户在一个视图中选择数据范围，其他关联视图同步更新，实现多模态数据的协同探索。

#### 核心技术落地

科学可视化、信息可视化、可视分析三大分支融合、多模态数据融合、多视图协调关联、跨维度数据关联表达。

### Astro-Insight visual strategy notes
- The platform should support choosing preferred color themes: warm / cool / dark / light.
- Let users choose chart preferences when multiple encodings are possible.
- Use filtering as an information-density control mechanism, not just a query tool.
- Aesthetic language should remain minimal, legible, and memorable.
- Color can start with grayscale foundations plus a single emphasized accent color.
- Animation should be used for transitions, temporal evolution, and motion-based storytelling, but only when it improves understanding.
- Familiar metaphors can help encode unfamiliar astronomical structures.
- Time visualization should support periodic layouts when the data are cyclic, and timeline layouts when the data evolve across years.
- Multimodal extensions may later combine structured data and audio, text, and annotations.
- The platform should explain why a result is credible, not only what the result is.
- If we compare with external datasets such as Kaggle or other public sources, we should be able to explain whether the conclusions align and why.
- The final presentation should include project background, dataset analysis, database E-R design, backend tech stack, and frontend tech stack.
